Output & Reports

infraudit supports multiple output formats for different workflows.

Console Output

Default output with colors and severity badges:

$ sudo infraudit audit

  infraudit v0.1.0 — Server Security Audit
  ══════════════════════════════════════════

  AUTH — Users & Authentication
  ─────────────────────────────
  ✗ CRITICAL  AUTH-001  SSH root login is enabled
                        → Set PermitRootLogin no in /etc/ssh/sshd_config
  ✓ PASS      AUTH-002  SSH password authentication disabled
  ✓ PASS      AUTH-003  No extra UID 0 accounts
  ! WARN      AUTH-006  3 sudoers entries use NOPASSWD
                        → Review /etc/sudoers.d/ and limit NOPASSWD usage

  NET — Network & Firewall
  ────────────────────────
  ✓ PASS      NET-001   Firewall is active (ufw)
  ✗ HIGH      NET-002   Ports 3306, 6379 open and not in whitelist
                        → Add to allowed_ports config or close with firewall

  ══════════════════════════════════════════
  Summary: 87 checks | 71 passed | 12 warnings | 4 failures
  Exit code: 2 (failures found)

Filtering Output

# Only show failures (no PASS results)
sudo infraudit audit --quiet

# Verbose — include all details for every check
sudo infraudit audit --verbose

# Combine: JSON failures only, save to file
sudo infraudit audit --format json --quiet --output failures.json

Exit Codes

For scripting and CI/CD integration:

# Use in CI pipeline
sudo infraudit audit --format json --output report.json
EXIT_CODE=$?

if [ $EXIT_CODE -eq 0 ]; then
    echo "All checks passed"
elif [ $EXIT_CODE -eq 1 ]; then
    echo "Warnings found — review recommended"
elif [ $EXIT_CODE -eq 2 ]; then
    echo "Failures found — action required"
    exit 1
fi
