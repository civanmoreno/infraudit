Configuration

Customize infraudit behavior with config files and CLI flags.

Config File Locations

infraudit looks for configuration in this order:

1. /etc/infraudit/config.yaml      # System-wide
2. ~/.infraudit.yaml               # User-level
3. ./infraudit.yaml                # Current directory
4. CLI flags                       # Highest priority

Example Configuration

# infraudit.yaml

# Server profile (web-server, db-server, container-host, minimal)
profile: web-server

# Output format (console, json, yaml)
format: console

# Skip specific checks by ID
skip:
  - HARD-007    # Swap check — this server uses unencrypted swap by design
  - NET-008     # IPv6 — we use IPv6 in this environment
  - SVC-012     # Desktop env — this is a jump box with GUI

# Skip entire categories
skip_categories:
  - container   # No Docker on this server
  - nfs         # No NFS in use

# Network: allowed open ports
network:
  allowed_ports:
    - 22        # SSH
    - 80        # HTTP
    - 443       # HTTPS

# Services: processes allowed to run as root
services:
  root_allowed:
    - sshd
    - nginx
    - fail2ban

# Resource limits thresholds
rlimit:
  disk_warn_percent: 85
  inode_warn_percent: 90

Server Profiles

Profiles pre-select relevant checks for common server roles:

ProfileDescriptionIncludes
minimal Only critical security checks CRITICAL and HIGH severity only
web-server Nginx/Apache web servers All + TLS/certificate focus, port 80/443 whitelisted
db-server Database servers (PostgreSQL, MySQL) All + credential file checks, network binding focus
container-host Docker/Podman hosts All + container category enabled

CLI Flags

FlagShortDescription
--version-vPrint version
--category-cRun checks for specific categories (comma-separated)
--profile-pUse a server profile
--format-fOutput format: console, json, yaml
--output-oWrite report to file
--skipSkip specific check IDs (comma-separated)
--verboseShow detailed output for each check
--quiet-qOnly show failures
